# OctoAcme Project Management Process

This repository documents OctoAcme's project management approach for cross-functional initiatives. It provides a clear, lightweight structure for initiating, planning, executing, releasing, and improving projects.

## Core approach

- Customer-first and outcome-driven
- Iterative delivery with small, testable increments
- Clear ownership for project, product, development, and QA roles
- Data-informed decisions with measurable success metrics
- Psychological safety through feedback and learning

## Key stages

1. **Initiation**
   - Define the problem, goals, stakeholders, and high-level timeline
   - Create a Project One-pager and verify sponsor alignment
   - Capture initial risks and decide whether to move to planning

2. **Planning**
   - Turn approved work into a prioritized backlog with acceptance criteria
   - Estimate effort, define the Definition of Done, and map milestones
   - Record dependencies and risks in a Risk Register

3. **Execution**
   - Build, test, review, and iterate on delivery
   - Keep the team aligned through regular standups and stakeholder syncs
   - Track progress against the release plan and update the backlog as needed

4. **Release**
   - Deploy outcomes, verify success, and communicate status
   - Confirm that acceptance criteria and quality checks are met

5. **Retrospective & improvement**
   - Capture learnings, document action items, and close the project
   - Feed feedback into future planning and process enhancements

## How to use these docs

- Use the project initiation, planning, execution, and retrospective docs as living guidance.
- Keep the artifacts updated in the repo so Copilot Spaces and teammates can access the latest process information.
